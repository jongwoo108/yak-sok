## 1. 약 변경 시 건강피드 자동 업데이트

### 1.1 Backend: Django Signal

- **파일**: `backend/apps/medications/signals.py` (신규)
- **내용**: `Medication` 모델의 `post_save` / `post_delete` 시그널 등록
- 약 생성/수정/삭제 시 `refresh_user_health_profile` Celery 태스크 자동 트리거
- GPT-4o 질병 재추론 → YouTube 검색 → 캐시 갱신 파이프라인 실행

- **파일**: `backend/apps/medications/apps.py`
- **변경**: `ready()` 메서드에서 signals 모듈 import 추가

### 1.2 Frontend: 자동 새로고침

- **파일**: `mobile/services/store.ts`
- **추가**: `healthProfileVersion` 카운터 + `invalidateHealthFeed()` 액션
- `createMedication`, `updateMedication`, `deleteMedication`, `deleteMedicationGroup` 성공 시 자동 호출

- **파일**: `mobile/app/(tabs)/health-feed.tsx`
- **추가**: `healthProfileVersion` 구독 → 변경 시 5초 대기 후 피드 자동 리로드
- Pull-to-refresh 시 `refreshProfile()` API 호출 → 4초 대기 후 피드 + 태그 갱신으로 강화

## 2. 역할 통합 (복약자 + 시니어 → 복약자)

기존 3역할(복약자/시니어/보호자)에서 2역할(복약자/보호자)로 통합.
코드 전체에서 복약자와 시니어가 100% 동일하게 처리되고 있었으므로 기능적 영향 없음.

> 상세 내용은 [10_role_structure.md](./10_role_structure.md) 참조

### 2.1 Backend 변경

| 파일 | 변경 |
|------|------|
| `apps/users/models.py` | `Role.SENIOR` 제거, `GuardianRelation.senior` FK `limit_choices_to` 수정 |
| `apps/users/views.py` | `[PATIENT, SENIOR]` → `PATIENT`로 단순화 |
| `apps/users/serializers.py` | default role `SENIOR` → `PATIENT` |
| `apps/medications/views.py` | `role__in` → `role=` 단순화 |

### 2.2 데이터 마이그레이션

- **파일**: `apps/users/migrations/0006_merge_senior_into_patient.py`
- 기존 `senior` 역할 사용자를 `patient`로 자동 변환
- `role` 필드 choices에서 `senior` 제거

```sql
-- 실행되는 SQL
UPDATE users_user SET role = 'patient' WHERE role = 'senior';
```

### 2.3 Frontend 변경

| 파일 | 변경 |
|------|------|
| `app/(auth)/register.tsx` | 역할 선택 3버튼 → 2버튼 (복약자/보호자) |
| `app/(tabs)/profile.tsx` | senior 역할 참조 제거, MESSAGE_OPTIONS 키 변경 |
| `services/types.ts` | `User.role` 타입에서 `'senior'` 제거 |
| `components/CustomTabBar.tsx` | 코멘트 정리 |

## 3. 배포

### 3.1 서버 (백엔드)

```bash
ssh -i ~/.ssh/LightsailDefaultKey-ap-northeast-2.pem ubuntu@3.39.142.149
cd /home/ubuntu/yak-sok
git pull origin main
docker-compose -f docker-compose.prod.yml down
docker-compose -f docker-compose.prod.yml up -d --build
docker-compose -f docker-compose.prod.yml exec web python manage.py migrate
```

```bash

npx eas-cli build --platform ios --profile production
```
